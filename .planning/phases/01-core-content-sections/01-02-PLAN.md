## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTML → External sites (GitHub links) | Project card "GitHub" links navigate to github.com — external origin, untrusted from our perspective |
| HTML → Image loading | Project images loaded from /images/projects/ — local static assets, trusted |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-03 | Spoofing | ProjectCard.astro (GitHub links) | mitigate | All GitHub links use target="_blank" rel="noopener noreferrer" to prevent tab-napping. URLs are hardcoded from CONTEXT.md — no user-controlled input. |
| T-01-04 | Information Disclosure | ProjectsSection.astro (image paths) | accept | Image paths are hardcoded static references. No sensitive paths exposed. |
| T-01-05 | Denial of Service | ProjectCard.astro (missing images) | mitigate | Images use loading="lazy" to avoid blocking page render. Placeholder bg-muted fallback if images missing per UI-SPEC image handling rules. |
| T-01-SC | Tampering | npm dependencies | mitigate | All packages [ASSUMED] in RESEARCH.md audit; no new installs. |
</threat_model>
