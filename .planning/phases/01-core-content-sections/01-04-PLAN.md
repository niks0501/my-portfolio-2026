    - meta name="description" content: "Full-stack developer portfolio showcasing web applications with clean UI and analytics-driven features. Available for freelance and contract work."
    - Keep: meta charset="utf-8", link rel="icon" (both svg and ico), meta name="viewport" content="width=device-width", meta name="generator" content={Astro.generator}

    **Body template:**
    Wrap all sections in a main element (semantic main landmark):
    ```html
    <body>
      <main>
        <Hero />
        <ProjectsSection />
        <AboutSection />
        <ServicesSection />
        <ContactFooter />
      </main>
    </body>
    ```

    **Component order per UI-SPEC section order (top to bottom):**
    1. Hero — headline, subline, email CTA, View Projects button
    2. ProjectsSection — heading + 3 project cards
    3. AboutSection — heading + paragraph
    4. ServicesSection — heading + 4-item list
    5. ContactFooter — Contact CTA heading + button, divider, footer links

    **Spacing between sections:** The components themselves carry their own py-16 vertical padding, so sections naturally space themselves. For About and Services, the UI-SPEC requires gap-8 (32px) between these contextually-related sections rather than the full py-16 doubling. A plain `mt-8` wrapper would produce ~160px (64px pb-16 from About + 32px mt-8 + 64px pt-16 from Services) — too much.

    The correct approach: wrap ServicesSection in a div that both provides the 32px gap AND overrides the component's top padding using a Tailwind arbitrary variant selector that targets the rendered `<section>` element:
    ```html
    <AboutSection />
    <div class="mt-8 [&>section]:pt-0">
      <ServicesSection />
    </div>
    ```
    This produces a 32px visual gap between the two sections' content areas: AboutSection's pb-16 (64px) + mt-8 (32px) = 96px of whitespace between About content end and Services content start. The `[&>section]:pt-0` selector targets the `<section>` element rendered by ServicesSection, overriding its `py-16` top padding to zero — collapsing the doubled section padding that would otherwise produce ~160px. Do NOT add `pt-0` directly on the wrapper div (it won't affect the child section). Do NOT use negative margins.

    **Language attribute:** html tag must have lang="en" (already present in existing boilerplate, keep it).

    **No JS imports, no client:load directives, no hydration.** Every component is pure static Astro.

    **No inline styles in index.astro.** All styling lives in the components themselves via Tailwind classes referencing global.css tokens.

    The existing boilerplate content (h1 "Astro", title "Astro") is fully replaced. Keep html/head/body structure but update all content.
